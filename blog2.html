Let's Go!

Introduction

Hello everyone! Welcome back to my blog where I dive into the inner workings of Bootstrap and explain my findings and research to you, the reader. In this blog post, I will work off what I had previously explained in my last blog post, where I gave you all a basic rundown on how to setup and use Bootstrap. In this blog post, I will be completely building a website from scratch using the tips and tricks I showed off in my previous demo to demonstrate the simplicity of creating websites using Bootstrap.

The Site

Foundations

To get things started, I needed to set a foundation for my website. I created a brand-new repository on GitHub to store all my work in, and to use GitHub Pages.



From there, I cloned the repository onto my PC so that I can work on it with my favourite IDE. With the foundation of my website set up, I am ready to code!



At this point, my repository is empty, so I need to populate it with something in order to publish it onto a GitHub Page, so I created a blank index.html file with Bootstrap's starter template inside it.



The Bread and Butter

With the foundation set, its time to code! The theme for my website is Formula 1, a motorsport that is touted as the pinnacle of motorsport, with drivers in F1 representing the best drivers in the world. With a theme in mind, I wanted to start with a navigation bar so that the user can navigate this website. Using Bootstraps official documentation, I added a navigation bar into my code.



I added a few more .html files so that my navigation bar actually brings you somewhere other than the index page. My next step is to style the navigation bar so that it matches the theme of Formula 1, because right now it looks very plain.



Now with a nice-looking navigation bar, lets spice up the home page a little bit because it is a blank canvas right now. I want to add a slideshow to the homepage to add some fluff to the home page, so I did just that. Once again, I grabbed a carousel component from the Bootstrap documentation and tweaked it to suit my page.



To add some more fluff, I added a BUNCH of cards to show off the drivers that currently populate the grid for the 2021 season. I could have made this part a lot more efficient somehow, but I digress.

I struggled a bit on this part because I wanted the driver cards to be side by side in columns of 2 but for whatever reason the cards would not line up the way I wanted them to. It had something to do with my < div class=”row row-mb-2> tag. row-mb-2 just means I am setting the margin-bottom(mb) of the row div to 0.5rem. (This is wrong.)

Eventually I got them how I wanted them by reviewing how the grid system works and how breakpoints divide the site up when viewed on a smaller screen, I simply removed the row-mb-2 and it worked!


They become stacked when viewed on a phone!

With the homepage looking somewhat presentable, I moved onto decorating the next webpage, the Racing Calendar. In here, I did the same thing to the track cards as I did with the driver cards. Duplicated them 23 times for all 23 races!


This block of code repeated 23 times... heh.

With the calendar populated, I moved onto the next page, which is just a simple informational page that briefly describes what Formula 1 is and how it works. Nothing too special. This page consists of a bunch of p tags with text describing Formula 1. Nothing new that you have not seen before.

The last element in my navigation bar is a dropdown menu showing all the teams that are participating in Formula 1 as of now. I color-coded each team with their respective colours using HTML to make them pop and stand out.


I was going to have each button redirect you to a team peage with more information on each team but... :^)

The dropdown menu has a cool animation that I coded using CSS. It is a simple fade-in fade-out animation that makes the website more visually appealing.

The Final Push

Now that this project has some heft to it, its time to push it to master and publish it online for you to see for yourselves. I used GitHub Desktop to do all my pushes for me because I like UI and pushing using CLI makes me sus.



Once the changes were pushed to master, all I had to do now was wait until my site updated with the changes I made and voila! My Formula 1 fan site is now online and visible to the world. Of course, since I made this with Bootstrap, you know this site is going to look nice on your mobile phones. (You guys all have phones, right?)
